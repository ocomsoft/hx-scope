hx-scope Extension Demo

Note: This demo uses a Service Worker to intercept requests and display the parameters being sent. You must serve this file from a web server (e.g., python -m http.server or npx serve) for the Service Worker to work. Opening the file directly (file://) will not work.

Demo 1: Basic CSS Class Selectors

Two separate forms on the same page. Each button uses a CSS selector to include only its inputs.

Demo 2: Multiple Selectors (Comma-Separated)

Use comma-separated CSS selectors to include inputs from multiple groups.

Demo 3: Attribute Selectors & Combining Selectors

Use attribute selectors and combine them with classes for powerful selection logic.

Demo 4: Checkbox hx-off-value Feature

Use hx-off-value to send a specific value when checkboxes are unchecked.

Note: Toggle the checkboxes and click "Save Settings" to see how:
  • Checked boxes send their value attribute
  • Unchecked boxes with hx-off-value send that value
  • Unchecked boxes without hx-off-value send nothing

Demo 5: Component Isolation - Line Item Calculation

This demonstrates the killer feature: Using hx-name (excluded from form) vs name (included in form) for component isolation.

Key Concept: Each line item uses hx-name="quantity" and hx-name="price" - the same names! These are NOT submitted with the form. Only the calculated totals (with name attribute) are submitted.

What's happening:
  • Calculate buttons: Use hx-scope to send only quantity and price from their specific line item to /calculate
  • Each line item: Uses the same hx-name="quantity" and hx-name="price" - no conflicts!
  • Checkout button: Only sends the hidden inputs with name attribute: line_total_1, product_id_1, line_total_2, etc.
  • NOT sent on checkout: The intermediate quantity and price values (they use hx-name, not name)

Try it: Change a quantity, click Calculate to update that line's total, then click Checkout to see only the final totals are submitted!
